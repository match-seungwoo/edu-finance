### ▷ 리스크 측정 (Week 4)

### Q41. 종목 가격을 첫날로 정규화한 뒤 합산하는 이유는? `prices.sum(axis=1)`은 왜 안 되나요?
**Check:** 가격 단위 상이(삼성전자 7만 vs LG화학 40만) → 단순 합은 비싼 주식이 NAV 지배 / 정규화로 "수익률 기준" 통일 후 비중 적용.

### Q42. VaR Historical/Parametric/Monte Carlo 중 하나를 골라 이유를 설명하면?
**Check:** Historical=분포 무가정(꼬리 반영, 데이터 의존) / Parametric=정규 가정(빠름, fat tail 과소) / MC=시뮬 1만회(유연, 비용) / 트레이드오프 설명 가능 여부.

### Q43. Historical VaR을 "가장 정직한 방법"이라 부르는 이유는?
**Check:** 분포 가정 없음 / `np.percentile(port_ret, 5)` 한 줄 / 데이터가 보여주는 꼬리를 그대로 반영 / 다만 과거에 없던 충격은 못 잡음.

### Q44. CVaR(Expected Shortfall)을 한 줄 코드로, 그리고 VaR보다 보수적인 이유는?
**Check:** `cvar = port_ret[port_ret <= var].mean()` / VaR=경계선, CVaR=그 너머 꼬리 평균 / 프로젝트 −4.2% < VaR −2.4% / fat tail에서 차이 큼.

### Q45. Parametric VaR 공식과 `−1.645`의 출처는?
**Check:** `VaR = μ + σ·norm.ppf(0.05)` / `norm.ppf(0.05)≈−1.645`=표준정규 5% 분위수 / 99%는 −2.326.

### Q46. Monte Carlo에서 `np.random.seed(42)`를 쓴 이유는?
**Check:** 재현성 — 동일 시드면 동일 결과 / 수업·디버깅·면접에서 비교 가능 / 실전은 시드 고정 + 다중 시드 평균 병행.

### Q47. 단순 정규 MC가 Parametric과 결과가 비슷한데도 MC를 배우는 진짜 이유는?
**Check:** 비선형·다자산·경로의존(옵션·barrier·MDD)엔 닫힌 공식 없음 / 분포 자유(Student-t 등) / 단순 케이스 일치는 "검증" 용도.

### Q48. 베타(β)는 무엇이고 어떻게 추정했나요?
**Check:** "시장 1% 변동 시 종목 몇 % 변동" / `scipy.stats.linregress(market, stock).slope` / β=1 시장 동행, β=1.5면 시장 −10%에 종목 −15%.

### Q49. KOSPI 응답 실패 시 합성 시장(5종목 평균)으로 fallback하는데, 그 약점은?
**Check:** 합성 시장이 실제 시장과 다름 / 5종목 평균은 그 5종목 평균 베타를 1로 강제(자기상관) / "MARKET(synthetic)" 라벨 명시 이유.

### Q50. 1요인 스트레스의 핵심 식과 프로젝트 결과는?
**Check:** `종목 PnL = β×시장충격`, `포트 PnL = Σ(wᵢ·βᵢ·shock)` / "KOSPI −15%" → 포트 −12.8%.

## 🔴 고급 (Hard) — 깊이·트레이드오프·설계 (Q56–Q85)

### ▷ 통계·수익률 이론

### Q56. 단순 수익률 대신 **로그 수익률** `ln(Pₜ/Pₜ₋₁)`을 쓰는 통계·시계열적 이유는?
**Check:** 시간 가법성(다기간 합=단순 덧셈) / 정규 근사 더 좋음 / 작은 값에선 단순 수익률과 거의 동일 / VaR/CVaR 모형 가정과 정합.

### Q57. 일일 변동성·수익률을 연환산할 때 √252를 곱하는 근거는?
**Check:** 분산은 시간에 비례(iid 가정) → 표준편차는 √t / 252 거래일 / 수익률은 ×252, 변동성은 ×√252 / 자기상관 있으면 가정 깨짐.

### Q58. 정규분포 가정의 Parametric VaR이 실제 데이터에서 위험을 과소평가하는 이유는?
**Check:** Fat tail(첨도>3)·왼쪽 치우침(skew)·변동성 군집(GARCH) / 1.645σ가 실제 5% 분위수보다 작음 / 해결: Student-t, EVT, Historical.

### Q59. "분산투자는 최악의 순간에 사라진다"를 ρ와 분산 공식으로 설명하면?
**Check:** 평시 ρ≈0.3→위기 ρ→0.9 / `σ_p² = Σwᵢ²σᵢ² + 2Σwᵢwⱼσᵢσⱼρᵢⱼ` / ρ→1이면 둘째 항 최대→포트 분산이 단순합에 수렴.

### Q60. 두 종목이 완전 양의 상관(ρ=1)이면 분산효과는? 프로젝트와 어떻게 연결되나요?
**Check:** 분산효과=0, 포트 VaR=비중가중 합 / 반도체 3종(삼성·SK·한미)은 ρ 높음(0.6~0.9) → "5종목이니 분산됐다"는 착각 경계 / W2 HHI 메시지의 정량화.

### Q61. CVaR가 "Coherent risk measure"인 이유와 VaR가 아닌 점은?
**Check:** CVaR는 subadditivity(분산이 위험을 늘리지 않음) 만족 / VaR는 일반적으로 위배 가능 / Basel 3 이후 ES(=CVaR)가 주 규제지표가 된 배경.

### ▷ 모델 검증·백테스팅

### Q62. VaR 백테스트(Kupiec)의 원리와, rolling 60일 VaR이 적정한지 어떻게 판정했나요?
**Check:** 60일 window로 5% VaR 추정→다음날 실손실이 더 깊으면 "위반" / 전체 위반률이 5%에 근접해야 적정 / 프로젝트 240일 중 12회=5%.

### Q63. `port_ret.rolling(60).quantile(0.05).shift(1)`에서 `.shift(1)`이 결정적인 이유는?
**Check:** shift 없으면 오늘 VaR을 오늘 손실과 비교 → lookahead bias / `.shift(1)`로 "어제까지 데이터로 추정한 VaR"을 오늘 손실과 비교 / 실제 운용 재현.

### Q64. 위반률이 너무 낮으면(<2%) 왜 문제인가요?
**Check:** 과도하게 보수적 → 필요 이상 자본 적립 / 자본 비효율·기회 손실 / VaR은 "너무 안전"해도 좋은 게 아님.

### Q65. 5종목·약 1년(250일) 데이터로 VaR을 추정하는 통계적 한계는?
**Check:** 5% 분위수에 표본 12~13개뿐→표준오차 큼 / 99% VaR은 2~3개→거의 무의미 / 실전: 5~10년, EWMA(시간가중), EVT(꼬리 극값).

### Q66. 베타·상관·민감도가 평시 데이터로 추정된다는 사실이 위기 분석에 만드는 한계는?

**Check:** "독자 연구"가 아닌 "built/designed/implemented (within a structured course)" / 단 능동적 검증·진단평가 통과는 진짜 증거라 강조 가능 / 정직성 원칙.

### Q97. "수익률 X% 달성" 같은 표현을 쓰면 안 되는 이유는?
**Check:** 이 프로젝트는 분석 시스템 구축이지 실제 투자 성과가 아님 / 백테스트≠실현 수익 / 과장은 신뢰 훼손 / "측정·시스템"을 성과로 표현.

### Q98. 이 경험이 지원하려는 전공/진로와 어떻게 연결되나요?
**Check:** (열린 질문) CS·데이터사이언스: AI 분업 설계·DB·파이프라인 / 금융공학·통계: VaR·MC·회귀 / 공통: "검증하는 습관" 서사 / 진정성 있는 연결.

### Q99. 다음 4주(5~8주차: Markowitz·Sharpe·Black-Litterman)에서 무엇을 가장 배우고 싶나요?
**Check:** W4 측정→W5 최적화 흐름 인지 / 위험 한도형 최적화 / 효율적 투자선·샤프 비율 / 측정에서 의사결정으로 넘어가려는 동기.

### Q100. 1분 안에 이 프로젝트 전체를 면접관에게 소개한다면?
**Check:** 한 줄 정의(AI 통합 엔드투엔드 퀀트 분석 시스템) / 4주 흐름(수집→집중도→DB→리스크) / 핵심 차별점("계산은 코드·해석은 AI·판단은 사람", 검증하는 습관) / 실수치 1~2개(MDD −28.5%, Kupiec 5%) / 30~60초 분량으로 압축.
