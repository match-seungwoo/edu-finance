문제 제기: 평균은 거짓말한다

2주차에서 본 연 평균 수익률 8%. 그것만 보고 들어갔다면?

  • 평균 8% 안에는 -40% 한 해도 들어 있다
  • 하락장은 몇 달 만에 누적 수익을 갉아먹는다
  • 투자자가 무너지는 건 평균이 아니라 최악의 순간
  • "버틸 수 있는가"는 평균이 아니라 꼬리가 결정한다

리스크 = 평균에서의 이탈이 아니다.

"리스크는 최악의 시나리오에서 살아남을 수 있는가이다. 그래서 평균이 아니라 분포의 꼬리를 본다."

핵심 개념: 리스크의 네 얼굴

변동성 (Volatility)

일일 수익률의 표준편차. 흔들리는 폭. 우상향 ETF도 이게 크면 멀미한다.

MDD (낙폭)

고점 대비 최대 손실폭. 경험한 가장 아픈 순간.

VaR / CVaR

확률적 최악. "95% 신뢰수준에서 잃을 수 있는 액수."

시나리오 충격

"만약 금리 +200bp가 한 번에 오면?" — 가정에 기반한 미래 손실.

MDD: Maximum Drawdown

"역사상 가장 깊은 골짜기까지 얼마나 내려갔나."

  • 고점(Peak)에서
  • 저점(Trough)까지의
  • 비율 손실

"평균은 -10%지만 MDD는 -45% — 중간에 못 버티고 던진다는 신호."

$\text{MDD} = \dfrac{\text{최저점 (Trough)} - \text{최고점 (Peak)}}{\text{최고점 (Peak)}} \times 100$

"고점에서 저점까지의 비율 손실(%)." 단, 저점은 그 고점 이후여야 한다 — 그래서 코드는 cummax(지금까지의 누적 최고점)를 쓴다.

1단계: MDD를 한 줄로

import pandas as pd # df["close"]: 종가 시계열 (DB에서 SELECT) nav = df["close"] / df["close"].iloc[0] # 누적 수익 곡선 peak = nav.cummax() # 지금까지의 최고점 drawdn = nav / peak - 1 # 현재 낙폭 (음수) mdd = drawdn.min() # 가장 깊은 골 mdd_date = drawdn.idxmin() # 그 날짜 print(f"MDD = {mdd:.2%} @ {mdd_date}")

cummax()는 "지금까지의 누적 최댓값". 이 한 함수가 MDD 계산의 핵심이다.

VaR: Value at Risk

"95% 신뢰수준 1일 VaR이 -2.4%다" 의 진짜 뜻:

  • 정상적인 100거래일 중 95일은 -2.4%보다 덜 잃는다
  • 나머지 5일은 -2.4%보다 더 잃을 수 있다
  • "최대 -2.4%까지 잃는다"가 아니다 (오해 1위)

VaR은 경계선이지 한계가 아니다. 5%의 꼬리는 늘 더 깊다.

$\text{VaR}_{\alpha} = -Q_{\alpha}(R)$

$Q_{\alpha}(R)$ = 수익률 분포의 $\alpha$-분위수.
$\alpha = 5\%$이면 분포 하위 5% 지점.

VaR을 구하는 세 가지 방법

① Historical

과거 수익률 분포의 5% 분위수를 그대로 사용. 가장 단순, 분포 가정 없음.

단점: 과거에 없던 충격은 못 본다.

② Parametric

수익률이 정규분포라고 가정. $\mu - 1.65\sigma$ (95%) 같이 닫힌 공식.

단점: 꼬리를 과소평가.

③ Monte Carlo

가정한 분포에서 1만 번 시뮬레이션. 분포·상관관계 자유롭게 설계 가능.

단점: 가정을 잘못하면 의미 없음.

"역사적 VaR로 시작해, 부족하면 시뮬레이션으로 보완한다."

2단계: Historical VaR (한 종목)

import numpy as np returns = df["close"].pct_change().dropna() var_95 = np.percentile(returns, 5) # 하위 5% 분위수 cvar_95 = returns[returns <= var_95].mean() # 그 너머의 평균 print(f"VaR 95% (1d) = {var_95:.2%}") print(f"CVaR 95% (1d) = {cvar_95:.2%}")

CVaR(=ES)는 "VaR 너머 5%의 평균 손실". VaR이 경계선이라면 CVaR은 그 너머 평균 — 꼬리의 두께를 본다.

3단계: Parametric VaR

from scipy.stats import norm mu = returns.mean() sigma = returns.std() # 정규분포 가정 — 95% 신뢰수준 var_95_param = mu + sigma * norm.ppf(0.05) print(f"Parametric VaR 95% = {var_95_param:.2%}")

$\text{VaR}_{95\%} = \mu - 1.645 \sigma$. 한 줄짜리 공식의 정체.

함정 — 실제 주가 수익률은 정규분포보다 꼬리가 두껍다. 2008·2020 같은 사건은 정규분포에선 "수만 년에 한 번."

4단계: Monte Carlo VaR

N = 10_000 sims = np.random.normal(mu, sigma, N) # 1만 번 가상 하루 var_mc = np.percentile(sims, 5) cvar_mc = sims[sims <= var_mc].mean() print(f"MC VaR 95% = {var_mc:.2%}") print(f"MC CVaR 95% = {cvar_mc:.2%}")

"미래를 직접 알 수 없으니, 가능한 미래 1만 개를 시뮬레이션해서 분포로 본다." 다자산·비선형 상품으로 확장하기 좋은 그릇.

5단계: 포트폴리오 단위 VaR

# 종목별 종가 → 일일 수익률 행렬 (열=종목) ret_mat = prices.pct_change().dropna() # 포트 비중 (1주차 가중치) w = np.array([0.4, 0.3, 0.2, 0.1]) # 포트폴리오 일일 수익률 시계열 port_ret = ret_mat @ w var_95_p = np.percentile(port_ret, 5) cvar_95_p = port_ret[port_ret <= var_95_p].mean()

"개별 종목의 VaR을 더한 값보다 포트 VaR이 작다" — 그 차이가 분산효과(diversification benefit)다.

6단계: 상관관계 — 분산의 진짜 엔진

"같이 떨어지면 분산이 아니다." 위기에는 상관계수가 1로 수렴한다.

  • 평시 $\rho \approx 0.3$
  • 위기 $\rho \to 0.9$
  • 분산효과는 가장 필요한 순간 사라진다
# 상관계수 행렬 — pandas 한 줄 ret_mat.corr() # 공분산 → 포트 분산 → 포트 표준편차 cov = ret_mat.cov() port_var = w @ cov.values @ w port_std = np.sqrt(port_var)

"분산이 만능이 아니다 — 위기일수록 $\rho$를 의심해야 한다."

7단계: 분포로 보는 VaR

  • 일별 수익률 히스토그램
  • 좌측 5% 위치에 VaR 수직선
  • 그 너머 영역을 색칠 → CVaR 영역
  • 학생이 "꼬리"라는 단어를 눈으로 이해한다
plt.hist(port_ret, bins=60) plt.axvline(var_95_p, color="red") plt.axvspan(min, var_95_p, alpha=0.2)

9단계: 시나리오를 포트에 입히기

# 종목별 베타 (시장 수익률에 대한 민감도) — 회귀로 이미 추정 betas = pd.Series({"005930": 1.10, "035720": 1.35, "068270": 0.85, "051910": 1.20}) market_shock = -0.15 # 코스피 -15% stock_shock = betas * market_shock # 종목별 충격 port_shock = (stock_shock * weights).sum() print(f"포트 충격 = {port_shock:.2%}")

베타 × 시장충격 × 비중의 합 — 가장 단순한 1요인 스트레스 모델. 실제는 금리·유가·환율 다요인으로 확장한다.

10단계: 다요인 충격 합산

# 종목 i의 충격 = Σ (factor_j 민감도 × factor_j 충격) sensitivities = pd.DataFrame({ "market": [1.10, 1.35, 0.85, 1.20], "rate": [-0.20, -0.30, 0.05, -0.10], # 금리 +1%당 "fx_usd": [0.40, 0.10, 0.50, -0.20], # USD +1%당 }, index=tickers) shocks = pd.Series({"market": -0.15, "rate": 0.02, "fx_usd": 0.10}) stock_returns = sensitivities @ shocks port_return = (stock_returns * weights).sum()

"민감도 행렬 × 충격 벡터 = 종목별 손익." 이게 사실상 모든 위험관리 시스템의 뼈대다.

11단계: 왜 AI에게 묻는가

지금까지는 분석가가 시나리오를 설계했다. AI에게 두 가지를 더 시킨다.

  • 시나리오 발굴 — "지금 시점에서 가장 위협적인 시나리오 3개"
  • 해석 — 숫자만 던지면 결론이 안 나온다. AI가 "이게 의미하는 바"를 정리
  • 도구 호출 — AI가 직접 stress_test() 함수를 부르게 한다

"분석은 사람이, 실행은 코드가, 해석은 AI가."

셋이 분업해야 의사결정이 빨라진다.

12단계: Function Calling 도구 정의

tools = [{ "name": "run_stress_test", "description": "포트폴리오에 거시 시나리오 충격을 적용한 손익을 계산", "input_schema": { "type": "object", "properties": { "market_shock": {"type": "number", "description": "코스피 충격, 예 -0.15"}, "rate_shock": {"type": "number", "description": "금리 변화 % 단위, 예 0.02"}, "fx_shock": {"type": "number", "description": "USD/KRW 변화율"}, }, "required": ["market_shock"], }, }]

AI는 자연어로 "금리 +200bp 시나리오 돌려줘"를 받아 → 위 함수로 구조화된 호출로 바꾼다.

13단계: AI ↔ 코드 핸드셰이크

import anthropic client = anthropic.Anthropic() resp = client.messages.create( model="claude-sonnet-4-6", max_tokens=1024, tools=tools, messages=[{ "role": "user", "content": "금리 +200bp, 코스피 -10% 시나리오로 내 포트 영향 알려줘" }], ) if resp.stop_reason == "tool_use": args = resp.content[-1].input result = run_stress_test(**args) # 우리가 만든 파이썬 함수 # 결과를 다시 AI에게 넘겨 자연어 요약 받기

제약을 명시할수록 AI 답이 단단해진다. "추측 금지"는 LLM 환각을 줄이는 가장 효과적인 한 줄.

16단계: 한계 — 모델은 모델이다

꼬리는 늘 더 두껍다

역사 데이터에 없는 사건은 VaR이 못 본다. "코로나 같은 일은 또 온다"는 가정.

상관관계는 변한다

평시의 $\rho$로 위기를 추정하면 손실을 과소평가한다.

AI는 숫자를 잘 못 한다

계산은 코드에, 해석만 AI에. 절대 AI에게 직접 평균·VaR을 묻지 않는다.

의사결정의 책임은 사람

AI는 시나리오 제안자·해설자. 매수·매도는 사람이 결정한다.

17단계: VaR 백테스트 (Kupiec 검정)

"VaR이 정말 95% 신뢰수준인가?" — 과거 데이터로 직접 세어 본다.

# 매일의 VaR 추정 vs 실제 손실 비교 (rolling window) exceptions = (port_ret < rolling_var_95).sum() expected = len(port_ret) * 0.05 print(f"실제 위반: {exceptions}회 / 예상: {expected:.0f}회")
  • 위반 ≈ 5% 근처면 모델 적정
  • 위반이 훨씬 많으면 → VaR 과소추정, 정규분포 가정 의심
  • 위반이 너무 적으면 → 보수적이지만 자본 비효율

핵심 메시지

  • 리스크는 평균이 아니라 꼬리다. MDD·VaR·CVaR로 그 꼬리를 수치화한다.
  • 통계만으로는 부족하다. 시나리오 = 이야기가 통계의 빈 칸을 메운다.
  • 분업의 원칙: 계산은 코드, 해석은 AI, 판단은 사람.
  • 모든 모델은 가정 위에 선다. 가정이 깨지는 그날을 가정 자체로 검증한다.
