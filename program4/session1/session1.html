Step 1 — 왜 '존재 확인'이 1번인가 🔍

분석가가 가장 많이 하는 실수: "데이터가 있겠지" 하고 분석부터 짠다.

파일이 없거나 · 비었거나 · 깨졌으면 → 그 위에 쌓은 모든 분석은 거짓말이 된다.

존재 → 개수 → 내용, 이 3단 확인을 첫날에 습관으로 박는다.

Step 1 — 코드: 파일 존재 + 크기 확인

need = ["corpus_clean.json", "corpus_clean.csv", "coverage_clean.csv", "manifest.json"] for f in need: p = os.path.join(backup, f) ok = os.path.exists(p) size = os.path.getsize(p) if ok else 0 print("✅" if ok else "❌", f, size, "bytes")

이 줄이 하는 일 존재만 보지 않고 bytes 크기까지 찍는다 — 0 bytes면 파일은 있어도 빈 파일이다.
