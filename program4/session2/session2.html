03

Step 2 — spaCy: 문장이 쪼개진다 ⭐

spaCy는 문장을 토큰으로 쪼개고, 각 단어에 역할 라벨을 붙인다. 오늘 필요한 건 딱 4개.

token.dep_

nsubj 능동 주어 · nsubjpass 수동 주어 · auxpass 수동 조동사

token.lemma_

시제 벗기기. attacked → attack, condemned → condemn

"Civilians were attacked" 를 파싱하면 — 누가 공격했는지가 문장에 없다. 이게 외교가 쓰는 기술이고, 오늘 셀 대상이다.

04

의존구문 한 번 읽기 — 같은 사건, 두 구조

두 문장을 파싱해 dep_ 라벨만 본다. 라벨이 책임의 위치를 말해준다.

토큰lemmadep_읽는 법
Russiarussiansubj능동 주어 — 행위자가 살아있다
attackedattackROOT본동사 (시제는 lemma로 벗긴다)
— "Civilians were attacked" —
Civiliansciviliannsubjpass수동 주어 — 당한 쪽

두 번째 문장에 nsubj(러시아)가 없다. 행위자가 문법적으로 증발 — directness가 이 신호를 센다. agent(by ...)까지 있어야 가해자가 복원된다 → Step 8 복선.

Step 3 — Directness Index (직설성) 개념

능동절 ÷ (능동절 + 수동절) − 우회표현 페널티. 결과는 0(우회적) ~ 1(직설적).

능동절

nsubj + head가 동사 → active += 1

수동절

nsubjpass · auxpass → passive += 1

우회표현

"it is regrettable" → 페널티 (상한 0.3)

페널티 상한 0.3인 이유: 우회표현 하나로 점수가 폭락하지 않게. 휴리스틱은 겸손해야 한다.

Step 3 — 능동/수동 판정 (코드 + CHECK)

def directness_index(text, nlp): doc = nlp(text) active = passive = 0 for tok in doc: # TODO: 수동 신호 두 가지 if tok.dep_ in ("nsubjpass","auxpass"): passive += 1 elif tok.dep_=="nsubj" \ and tok.head.pos_ in ("VERB","AUX"): active += 1 base = active/(active+passive) return base - penalty # 상한 0.3

CHECK — 두 문장으로 검증

"Russia attacked civilians" → active 1, passive 0 → 1.0

"Civilians were attacked" → passive ≥ 1 → 0.0

Step 4 — Verb Strength Ladder (동사 강도)

외교 동사의 강도계. note(봤다)부터 demand(요구한다)까지 7단계.

1234567
note
봤다		 acknowledge regret concern deplore condemn
규탄		 demand
요구

각 단계에 동의어가 묶여있다 — 5에 deplore·reject·oppose, 6에 condemn·urge, 7에 demand·insist.

max = 한 번이라도 강했나 · mean = 전반적 톤. 둘을 함께 본다.

08

Step 4 — 동사 사다리 매핑 (lemma 매칭)

def verb_strength(text, nlp): doc = nlp(text); hits = [] for tok in doc: lemma = tok.lemma_.lower() if lemma in VERB_LADDER \ and tok.pos_ in ("VERB","NOUN","ADJ"): hits.append(VERB_LADDER[lemma]) return {"max": max(hits), "mean": sum(hits)/len(hits)}

lemma 매칭이 핵심

condemned → lemma condemn → 6점

condemnation(명사) → 같은 lemma → 잡힘

시제·품사가 달라도 기본형으로 맞춘다. 그래서 pos_ in (VERB,NOUN,ADJ) 까지 채집.

복선: verb_strength_max는 길이에 약하다 — 긴 문서일수록 강한 동사 한 번 나올 확률↑ → Step 7·S4 재경고.

Step 5 — Subject Pattern (주어 패턴) 개념

"누가 주어 자리에 서나"를 6범주로 분류한다.

first_person

we / our / us

national_actor

대한민국 / 외교부 / Russia

intl_community

the international community

all_parties

all parties / both sides

+ impersonal(it/there) · other. "We condemn"(주체적)와 "대한민국은"(기관 목소리)은 같은 규탄이라도 거리감이 다르다.

Step 8 — SVO 추출 + 경량 coreference ⭐

세는 것에서 → 뽑는 것으로

지금까지 라벨을 세기만 했다. 이제 한 동사 중심으로 주어-동사-목적어(SVO) 삼항을 통째로 뽑는다.

from diplo_analysis import ( extract_svo, blame_attribution) # "Russia attacked civilians" # → subj=Russia passive=False (named) # "Civilians were killed" # → subj=None passive=True (obscured)
  • 새 라벨 agent = 수동문 "by ..." 행위자 (killed by Russia) → nsubj/nsubjpass와 한 표로 본다
  • 경량 coref: "It bombed the city" 의 it 같은 대명사를 앞 문장의 국가로 되돌린다 — SVO+coref는 길어서 검증된 툴킷 재사용
  • 귀속은 가해를 서술한 문장에서만 발동 → 희소 → 우크라이나+가자 합쳐 표본 키움

긴 문서일수록 강한 동사가 한 번이라도 나올 확률이 올라간다. max는 길이에 약하다.

  • 중국 브리핑은 길다 → max 6.20이 강경함인지 길이인지 모른다

복선 회수

Step 4에서 깐 복선이 여기서 회수된다 — "max는 길이에 약하다"가 표의 1위(중국)를 흔든다.

mean을 함께 보면 길이에 덜 휘둘린다 — 두 지표를 항상 같이 읽는 이유.

흔한 함정 ② — 귀속이 sparse한 이유 · 복문 누락

왜 ~101건뿐인가

귀속은 가해를 구체적으로 서술한 문장에서만 발동한다. 추상적 규탄("깊이 우려")에는 가해 동사가 없어 0건.

→ 절대 수치 말고 소스 간 상대 순서를 신뢰한다. 한국 0건은 버그가 아니라 발견이다.

복문 누락

소형 spaCy 모델(en_core_web_sm)은 복문에서 일부 SVO를 놓친다.

"러시아가 …하고 도시가 폭격됐다" 같은 절은 한쪽 삼항이 빠질 수 있다 → 카운트 과소 가능.

규칙이 못 잡는 framing은 다음 주 LLM이 교차검증.
