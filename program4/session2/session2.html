01

한 문장: 러시아는 어디로 사라졌나

능동 — 행위자가 살아있다

"Russia attacked civilians"

러시아가 주어. 누가 했는지 명확.

수동 — 행위자가 증발했다

"civilians were attacked"

러시아가 사라졌다. 누가 했는지 없음.

같은 사건, 다른 문장 구조. 외교 성명은 이 차이로 책임을 흐린다. 오늘 그걸 숫자로 잡는다.

02

Step 2 — spaCy: 문장이 쪼개진다 ⭐

spaCy는 문장을 토큰으로 쪼개고, 각 단어에 역할 라벨을 붙인다. 오늘 필요한 건 딱 4개.

token.dep_

nsubj 능동 주어 · nsubjpass 수동 주어 · auxpass 수동 조동사

token.lemma_

시제 벗기기. attacked → attack, condemned → condemn

"Civilians were attacked" 를 파싱하면 — 누가 공격했는지가 문장에 없다. 이게 외교가 쓰는 기술이고, 오늘 셀 대상이다.

03

Step 3 — Directness Index (직설성)

아이디어 한 줄

능동절 ÷ (능동절 + 수동절) − 우회표현 페널티. 결과는 0~1.

  • 수동절 감지 = nsubjpass / auxpass
  • 우회표현 "it is regrettable" → 페널티 (상한 0.3)
  • 능동 1.0 vs 수동 0.0 이면 통과

페널티 상한 0.3 — 휴리스틱은 겸손해야 한다.

def directness_index(doc): active = passive = 0 for tok in doc: if tok.dep_ == "nsubj": active += 1 # TODO: 학생이 채움 if tok.dep_ in ("nsubjpass","auxpass"): passive += 1 return active / (active + passive)

외교 동사의 강도계. lemma 매칭이라 condemned 같은 과거형도 잡힌다.

+ urge / reject / oppose 도 사다리에 포함. max(한 번이라도 강했나) vs mean(전반적 톤)을 함께 본다.

복선: max는 길이에 약하다. 긴 문서일수록 강한 동사 한 번 나올 확률↑ → Step 7·S4에서 길이 통제로 다시 경고.

주어를 6범주로 분류. 명사구는 left_edge ~ right_edge 로 통째로 잡는다.

national_actor

intl_community

all_parties

07

Step 8 — SVO 추출 → Blame Attribution (귀속) ⭐

9개 차원 중 8번 — 첫 등장

가해 동사(attack/strike/kill)에서 가해자가 누구로 채워졌나를 본다.

# 검증된 툴킷 재사용 from diplo_analysis import ( extract_svo, blame_attribution) # Russia attacked civilians # → subject=Russia (named) # Civilians were killed # → subject=None (obscured)

새 라벨 agent = 수동문 "by ..." 행위자. 희소해서 우크라이나+가자 합침.

per-doc 평균 blame_directness

FR 0.43 > UN 0.38 > CN 0.30

한국 = 귀속 0건

  • 한국: 공격을 서술 안 함 — "깊이 우려·규탄"식 추상적 규탄만 → 가해 동사 0
  • 중국: 행위자 가리는 수동태 비율 최고 → 가장 많이 가림(0.30)
  • 프랑스 주의: 발동 7건뿐 — 절대수치보다 구조 신호
